Comments
    used to explain python code.
    used to make the code more readable.
    used to prevent executiong when testing code.
    starts with a #, and python will ignore them.
    can be placed at the end of the line, and python will ignore the rest of the line.
    doesn't have to be text that explains the code, can also be used to prevent python from executing code.
    doesn't really have syntax for multi-line comment. or not quite as intended, we can use multi-line string. Since, Python will ignore string literals that are not assigned to a variable, we can add a multi-line string(triple quotes) in our code and place our comment in it.

Getting Started with FASTAPI
    What is FastAPI?
        mordern, fast(high-performance), web framework for building APIs with python based on standard python type hints.
        Key features:
            Fast: Very high performance, on par with NodeJS and GO(thanks to Starlette and Pydantic)
            Fast to cod: Increase the speed to develop features by about 200% to 300%
            Fewer bugs: Reduce about 40% of human(developers) induced errors
            Intuitive: Great editor support, Completion everywhere, less time debugging
            Easy: Designed to be easy to use and learn, less time reading documents
            Short Minimize code duplication, Multiple features from each parameter duplication, Fewer bugs
            Robust: Get production-ready code, with automatic interactive documentation
            Standard-based: Based on(and fully compatible with) the open standards for APIs: OpenAPI(previously known as swagger) and JSON schema
        FastAPI stands on the shoulder of giants:
            Starlette for the web parts 
            Pydantic for the data parts
        
Installing FastAPI and virtual enviroment installation (Window, Mac and Linux)
    Setting up virtual environment 
        python -m venv venv 
        the above code snippet is written as CLI in the terminal of the project directory. Then
        ".\venv\Scripts\activate" for windows and "source venv/bin/activate" for MacOS and Linux to start or activate the virtual environment.
        pip freeze or pip list can be used to see the installed package in the virtual enviroment
    Installation of FastAPI can be done in 2 ways;
        pip install fastapi 
            This install only the core FastAPI package:
                includes the main frameworks
                depends on essentials like Starlette and Pydantic
                doesnot include extras for running a full production application
                we will need to manually install things like:
                    ASGI server (eg: Uvicorn)
                    Performance tools
                    Optinoal features
        pip install "fastapi[standard]"
            This install fastAPi plus a bundle of commonly used extras
            The [standard] extra typically includes:
                Uvicorn (to run our application)
                Uvloop (faster async performance, Unix only)
                httptools (better performance)
                python-multipart (for file uploads)
                jinja2 (if we use templates)
            In short, it gives us a ready to run, production-friendly setup without installing each piece manually
        When to use which?
            If we want full control over dependencies and are building minimal or custom setup using fastapi is enough.
            If we want to start quickly and are building a typical weh API using fastapi[standard] is prefered.

What is uvicorn?
    a fast ASGI server used to run python web application like those built with FastAPI
    a server that runs our FastAPI application
    without uvicorn, our FastAPI code won't actually be accessible in browser or via API calls
    Why uvicorn is needed?
        when we write a FastAPI application, we're just defining routes and logic.
        Uvicorn:
            starts a web server
            Listens for incoming request(Like GET/POST)
            sends responses back to the client 
    What is ASGI(Asynchronous Server Gateway Interface)?
        supports async/await
        handles many request efficiently
        mordern alternative to WSGI(used by older frameworks)
    Runs using: uvicorn main:app --reload or fastapi dev main.py

        python has support for optional "type hints"(also known as "type annotations")
        These "type hints" or annotations are a special syntax that allows declaring the type of a variable 
        By declaring types for our variables, editors and tools can give us better support
        FastAPI is all based on these type hints, they give it many advantages and benefits.
        Type hints are mainly declared as the function parameter.
    Type hints in FastAPI
        FastAPI takes advantages of these type hints to do several things
        with FastAPI, we declare parameters with type hints and we get:
            Editor support 
            Type check 
        ___ and FastAPI uses the same declaration to :
            Define requirements: from request path parameters, query parameters, headers, bodies, dependencies, etc.
            Convert data: from request to required type
            validate data: coming from each request
                Generating automatic errors returned to the client when the data is invalid
            Document the API using OpenAPI
                which is then used by the automatic interactive documentation user interfaces.

Concurrency and async/await
    Concurrency is the concept of dealing with multiple tasks at once. It doesn't neccessarily means they are being executed at the exact time but rather, they are managed in a overlapping manners. Threads and coroutines are used to achieve concurrency.
    async/await are keywords used in python to handle asynchronous programming. This allows the program to perform other tasks while waiting for long-running tasks(like network calls or file I/O) to complete, instead of blocking the whole process.
    async: Marks a functions as asynchronous, meaning it can be paused and resumed later.
    await: Used to call an asynchronous function and tells python to "pause" and wait for that function to finish without blocking the rest of the code.
    Key difference between concurrency and parallelism 
        concurrency: managing multiple tasks at once(task may or may not run at the same time)
        parallelism: Actually running multiple tasks simultaneously(requires multiple CPU cores or threads)
    Use of async/await 
        If we are using third party libaries that tell us to call them with await. Then, declare our path operation function with async function with async def 
        If we are using third party libaries that communicates with something(like database, an API, the file systme, etc) and doesn't have support for using await, then declare our path operation functions as normally with just def 
        If our application(somehow) doesn't have to communicates with anything else and wait for it to respond, use async def, even in case of no usage of await inside
        (Note: We can only use await inside async functions)
    
    Starlette and FastAPI are based on AnyIO which makes it compatible with both python standard libary asyncio and trio.
    AnyIO can be used directly for our advanced concurrency use cases that require more advanced patterns in our code.
    with the use of AnyIO, we can write our own async application without using FastAPI.
    Coroutine is just the very fancy term for the thing returned by an async def function
    Mordern versions of python support for "asynchronous code" using something called "coroutines", with async and await syntax.

URL(Uniform Resource Locator)
    is the address used to access a resource on the web
    structure of URL: schema://domain:port/path?query#fragments
        Schema: defines the protocols used to access resource
            Example: http, https, ftp ----> https://
        Domain(Host): Human-readable name that identifies the server where the resource is hosted
            Example: www.example.com 
        Port(optional): Numbers that identifies the specific service on the server to connect to. If not specified, the default port for the schema is used.
            Example: 80 for HTTP; :443 for HTTPS 
        Path: Specific location of resource on the server 
            Example: /blog/article 
        Query(optional): Provides additional data to the server, typically in form of key-value pairs.
            Example: ?search=python&limit=5
        Fragment(optional): Reference to a specific section of the resource(often a specific part of a webpage)
            Example: #section1  or #comments 

What is decorator?
    @something syntax in python is called "decorator"
    we put it on top of function, like a pretty decorative had(that's where the term came from)
    Takes the function below and does something with it.
    In case of FastAPI, the decorator tells that the function below corresponds to the path "/path_routes" with an operation "HTTP_methods".
    It is called as the path operation decorator.
