Getting Started with FASTAPI
    What is FastAPI?
        mordern, fast(high-performance), web framework for building APIs with python based on standard python type hints.
        Key features:
            Fast: Very high performance, on par with NodeJS and GO(thanks to Starlette and Pydantic)
            Fast to cod: Increase the speed to develop features by about 200% to 300%
            Fewer bugs: Reduce about 40% of human(developers) induced errors
            Intuitive: Great editor support, Completion everywhere, less time debugging
            Easy: Designed to be easy to use and learn, less time reading documents
            Short Minimize code duplication, Multiple features from each parameter duplication, Fewer bugs
            Robust: Get production-ready code, with automatic interactive documentation
            Standard-based: Based on(and fully compatible with) the open standards for APIs: OpenAPI(previously known as swagger) and JSON schema
        FastAPI stands on the shoulder of giants:
            Starlette for the web parts 
            Pydantic for the data parts

Concurrency and async/await
    Concurrency is the concept of dealing with multiple tasks at once. It doesn't neccessarily means they are being executed at the exact time but rather, they are managed in a overlapping manners. Threads and coroutines are used to achieve concurrency.
    async/await are keywords used in python to handle asynchronous programming. This allows the program to perform other tasks while waiting for long-running tasks(like network calls or file I/O) to complete, instead of blocking the whole process.
    async: Marks a functions as asynchronous, meaning it can be paused and resumed later.
    await: Used to call an asynchronous function and tells python to "pause" and wait for that function to finish without blocking the rest of the code.
    Key difference between concurrency and parallelism 
        concurrency: managing multiple tasks at once(task may or may not run at the same time)
        parallelism: Actually running multiple tasks simultaneously(requires multiple CPU cores or threads)
    Use of async/await 
        If we are using third party libaries that tell us to call them with await. Then, declare our path operation function with async function with async def 
        If we are using third party libaries that communicates with something(like database, an API, the file systme, etc) and doesn't have support for using await, then declare our path operation functions as normally with just def 
        If our application(somehow) doesn't have to communicates with anything else and wait for it to respond, use async def, even in case of no usage of await inside
        (Note: We can only use await inside async functions)
    
    Starlette and FastAPI are based on AnyIO which makes it compatible with both python standard libary asyncio and trio.
    AnyIO can be used directly for our advanced concurrency use cases that require more advanced patterns in our code.
    with the use of AnyIO, we can write our own async application without using FastAPI.
    Coroutine is just the very fancy term for the thing returned by an async def function
    Mordern versions of python support for "asynchronous code" using something called "coroutines", with async and await syntax.

URL(Uniform Resource Locator)
    is the address used to access a resource on the web
    structure of URL: schema://domain:port/path?query#fragments
        Schema: defines the protocols used to access resource
            Example: http, https, ftp ----> https://
        Domain(Host): Human-readable name that identifies the server where the resource is hosted
            Example: www.example.com 
        Port(optional): Numbers that identifies the specific service on the server to connect to. If not specified, the default port for the schema is used.
            Example: 80 for HTTP; :443 for HTTPS 
        Path: Specific location of resource on the server 
            Example: /blog/article 
        Query(optional): Provides additional data to the server, typically in form of key-value pairs.
            Example: ?search=python&limit=5
        Fragment(optional): Reference to a specific section of the resource(often a specific part of a webpage)
            Example: #section1  or #comments

Request Body 
    When we need to send data from client(let's say, a browser) to our API, we send it as a request body.
    A request body is data sent by the client to our API. A response body is the data our API sends to the client.
    Our API almost always has to send a response body, but client doesn't necessarily have to send request body.
    To declare a request body, we use Pydantic models with all their power and benefits.
    To send data, we should use one of the POST(the most common), PUT, DELETE, PATCH
    Sending a request body with a GET request has an undefined behaviour, nevertheless, it is supported by FastAPI, only for very complex/extreme use cases.

Query Parameters and String Validations
    FastAPI allows us to declare additional information and validation for our parameters.
    = None shows that default query is None so, FastAPI will know it's not reqiured and othe types hints like str and None helps to get better support and detect errors.
    For additional string validations:
        Example: Length doesn't exceed 50 characters
            First of all import Query from fastapi and Annotated from typing
            q: Annotated[str | None, Query(max_length = 50)] = None  or q: str | None = Query(default=None, max_length=50) is used in older codebase
            FastAPI will now:
                validate the data making sure that max length is 50 characters.
                shows clear error for the client when the data is not valid. 
                Document the parameter in OpenAPI schema path operation(so it will show up in the automatic docs UI)
            Using Annotated is recommended instead of the default value in function parameters, it is better in multiple reasons:
                The default value of the function parameter is the actual default value, that's more intuitve with python in general.
                We could call the same function in other places without FastAPI, and it would work as expected. If there's a required parameter(without a default value), our editor will let us know with an error, Python will also complain if we run it without passing the required parameter.
                When we don't use Annotated, And instead use the (old) defauld value style. If we call that function without FastAPI in other places, we have to pass the arguments to the function for it to work correctly, otherwise the values be different from what we expect(eg: QueryInfo or something similar instead of str). And our editor won't complain, and python won't complain running that function, only when the operation inside error out.

Bodies of pure lists 
    When i tried to post a list of image using(def func(images: list[Image])), it showed "{detail: "Not Found"}" error. This error was showcased because FastAPI reads the body using Starlette's sync request parsing, which block the event loop while reading JSON, So it doesn't parse the body properly in case of large JSON arrays.
    Then i tried using async and it works reliably, because FastAPI awaits the JSON parsing properly.
    Why async def fixes it?
        FastAPI internally calls await request.json() when parameter type is complex(Like list[Image]).
        If our endpoint is sync, it uses a "sync wrapper" around request.json() -> may fails on lists at top-level or with certain clients(Postman sometimes triggers this).
        If our endpoint is async, FastAPI directly awaits the body parsing, so it works reliably.
    Quick rule of Thumb:
        Use asunc def when our endpoint:
            Accepts raw lists or large JSON bodies 
            Uses nested models
            Might perform I/O operations later 
        Use def for simple, small, synchronous endpoint 

(Note: * on def func(*, user_id: int = ...) helps to force user_id to be keyword-only. without *, Python would allow us to accidentallu call it. with *, client request /users/123/ , FastAPI matches 123->user_id, so we con't call read_user(123))
